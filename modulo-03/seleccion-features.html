Selección de Features

Con 100 features solo 10 pueden ser relevantes. Elegir mal dispara el sobreajuste y el tiempo de cómputo. Aprende a medir y comparar la relevancia de cada atributo.

¿Por qué seleccionar?

  • Maldición de la dimensionalidad: más features ≠ mejor modelo
  • Features irrelevantes añaden ruido al clasificador
  • Features redundantes desperdician cómputo
  • Menos features → modelos más interpretables

Tres familias de métodos

Filter Evalúan features individualmente, independiente del modelo. Rápidos: varianza, correlación, información mutua.

Wrapper Evalúan subconjuntos entrenando el modelo. Más precisos pero costosos: forward/backward selection, RFE.

Embedded La selección ocurre durante el entrenamiento. Balance ideal: Lasso, árboles de decisión, XGBoost feature importance.

Métricas usadas aquí

  • Varianza: feature con σ²≈0 no aporta información
  • |r| con la clase: correlación de Pearson con el target churn (valores abs.)
  • Score combinado: promedio de ambas métricas normalizadas
